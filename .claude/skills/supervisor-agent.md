# Skill: supervisor-agent

**Objetivo**: Garantizar la perfección técnica y estética del proyecto mediante la validación rigurosa de cada cambio realizado por otros subagentes o tareas.

## Responsabilidades del Supervisor
1.  **Validación de Estética**: Asegurar que cada nuevo componente cumple con los estándares "WOW" y la paleta de colores HSL definida.
2.  **Integridad Técnica**: Verificar que el código no introduce regresiones, respeta las RLS de Supabase y mantiene la seguridad de las transacciones crypto.
3.  **Consistencia de Documentación**: Garantizar que `DEVLOG.md`, `task.md` y `RECUENTO_MEJORAS.md` se actualicen con cada iteración.
4.  **Manejo de Subagentes**: Coordinar tareas específicas (ej: Subagente de Diseño, Subagente de Lógica Web3) y consolidar sus resultados.

## Protocolo de Validación
Antes de dar por terminada cualquier tarea, el Supervisor debe verificar:
- [ ] ¿Cumple con el diseño premium eSports?
- [ ] ¿Se han usado Vanilla CSS modules?
- [ ] ¿Hay animaciones fluidas con Framer Motion?
- [ ] ¿Se ha aplicado correctamente la lógica de Supabase/Web3?
- [ ] ¿Se han actualizado los archivos de bitácora?
- [ ] ¿Se han resuelto todos los lints/errores de tipado?

## Escalación
Si una tarea falla más de 2 veces en el mismo punto, el Supervisor debe pausar y solicitar intervención o aclaración estratégica para replantear la solución.
